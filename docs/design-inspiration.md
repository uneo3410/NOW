# Now 时间线：设计灵感文档

## 1. 项目直觉

Now 时间线的核心不是“管理任务”，而是记录一个人如何从念头、行动、完成，逐渐形成自己的生活轨迹。

传统 Todo 的问题是：

- 它太线性。
- 它默认人生是一张清单。
- 它会把生活压成 KPI。
- 未完成的东西会不断堆积，变成心理噪音。

但真实的大脑不是这样工作的。

真实的大脑更像一张网：

- 一个念头牵出另一个念头。
- 一个计划旁边会长出灵感。
- 一个今天想做的事，可能最终变成时间线上的一个生活节点。
- 很多重要的事并不是“任务”，只是发生了，值得留下。

所以这个项目应该更像：

> 一个会把“现在”沉淀下来的个人时间系统。

---

## 2. 两个核心空间

### 2.1 时间线页面

时间线是沉淀层。

它负责记录：

- 已经发生的事
- 已经完成的事
- 值得留下的瞬间
- 手动补录的生活节点
- 从 Todo 卡片完成后自动生成的节点

时间线不是任务列表。

它更像一个“生活网格”或“人生轨迹切片”。

### 2.2 思维画布页面

思维画布是流动层。

它负责承载：

- 想法
- 灵感
- 今日想做的事
- 暂时不知道归类的念头
- 可以互相连接的卡片
- 可完成的 Todo 卡片

它不要求线性排序。

用户可以把卡片拖来拖去、拉线、形成链条，也可以单纯把它当成一张思维导图。

---

## 3. 两种卡片

### 3.1 普通思维卡片

普通卡片没有完成状态。

它可以是：

- 一个概念
- 一个灵感
- 一个问题
- 一段备注
- 一个未来可能会做的东西

它存在的意义是帮助思考，而不是催促完成。

### 3.2 Todo 卡片

Todo 卡片是带完成状态的特殊卡片。

视觉上可以带一个小圆圈或小勾。

当用户点击完成：

1. 卡片在画布中淡出 / 消散。
2. 系统记录完成时间。
3. 时间线页面自动生成一个时间节点。
4. 节点内容使用原卡片文字。

例子：

> 画布中创建 Todo 卡片：“看一部电影”  
> 看完后点击完成  
> 卡片消散  
> 时间线自动出现节点：“看一部电影” + 当前时间

---

## 4. 时间线节点的来源

时间线节点应该支持多种来源。

不要只允许 Todo 卡片流入时间线。

因为很多值得记录的东西不是任务，它们只是发生了。

建议保留两种入口：

### 入口 A：时间线主动创建

用户在时间线页面点击某个位置，主动创建一个时间节点。

适合：

- 记录已经发生的事
- 补录今天的片段
- 记录梦、灵感、情绪、事件
- 不属于 Todo 的生活瞬间

### 入口 B：Todo 卡片完成后自动创建

用户在画布完成 Todo 卡片，系统自动创建时间线节点。

适合：

- 从“想做”到“做完”的行动闭环
- 把完成的事情自动沉淀为生活记录

---

## 5. 设计原则

### 5.1 底层统一，入口分开

UI 上可以有两个入口：

- 时间线页主动创建
- 画布 Todo 完成生成

但底层数据应该统一成同一种 Timeline Node。

示例：

```ts
type TimelineNode = {
  id: string
  content: string
  createdAt: string
  happenedAt: string
  source: "manual" | "todo-card" | "import" | "system"
  sourceCardId?: string
}
```

这样未来查询、同步、导出、统计都会简单很多。

### 5.2 不要把它做成传统效率工具

这个项目不应该充满：

- 优先级
- 截止日期
- 红色警告
- 连续打卡
- 完成率统计
- KPI 风格图表

至少 MVP 阶段不要。

它应该更像一个温和但有结构的个人生活系统。

核心不是逼迫用户完成更多，而是帮助用户看见自己正在如何生活。

### 5.3 本地优先

MVP 优先本地可用。

推荐路径：

1. PWA + IndexedDB
2. 支持 iOS Safari 添加到主屏幕
3. 后续再考虑同步
4. 再后续才考虑封装 iOS / Android

---

## 6. PWA 路线判断

当前不建议首发原生 App。

原因：

- iOS App Store 上架麻烦
- 审核、证书、开发者账号都有额外成本
- 当前项目还在概念探索期，不适合过早工程重型化
- PWA 足够做出个人 MVP

推荐技术路线：

- 前端：React / Vue / Svelte 均可
- 画布：React Flow / Vue Flow / 自研轻量画布
- 存储：IndexedDB，外层可用 Dexie
- PWA：manifest + service worker
- 后续封装：Capacitor 优先

---

## 7. MVP 功能草案

### 时间线页

- 创建节点
- 编辑节点
- 删除节点
- 按时间展示
- 显示来源：手动 / Todo 完成

### 画布页

- 创建普通卡片
- 创建 Todo 卡片
- 拖拽卡片
- 连接卡片
- 完成 Todo 卡片
- 完成后生成时间线节点

### 数据层

- Card 表
- Edge 表
- TimelineNode 表
- 本地持久化

---

## 8. 一句话定义

Now 时间线是一个把“想法、行动和已经发生的生活”连接起来的个人 PWA。

它不是 Todo。

它是：

> 从现在开始，给生活留下形状。
