# Now 时间线：付费级 UI/UX 标准

## 1. 核心目标

Now 时间线即使早期免费，也要按照“付费产品”的完成度来设计 UI/UX。

这里的付费标准不是指一开始就收费，而是指：

> 用户打开它时，会觉得这是一个完整、顺滑、有审美、有记忆点的产品，而不是一个功能拼出来的 demo。

先把能让人愿意买单的部分做出来，再考虑商业模式。

---

## 2. 用户愿意买单的地方

### 2.1 视觉质感

这个项目的核心体验是时间、卡片、画布和回顾。

所以它必须有明确的视觉气质：

- 干净
- 克制
- 有空间感
- 有轻微仪式感
- 不像普通效率工具
- 不像后台管理系统
- 不像 Notion 插件

目标不是“功能很多”，而是“我愿意每天打开”。

### 2.2 交互手感

付费感很大一部分来自手感。

重点动作：

- 创建时间点
- 创建卡片
- 拖动卡片
- 连接卡片
- 完成 Todo
- 时间线缩放
- 画布缩放
- 切换桌面 / 移动视图

这些动作必须顺。

如果 Todo 完成后卡片消失、时间线生成节点这一瞬间做得好，它会成为产品记忆点。

### 2.3 情绪反馈

传统 Todo 的反馈是：

> 完成了，划掉。

Now 时间线的反馈应该是：

> 完成了，它成为了你生活的一部分。

所以动效不应该只是 checkbox 状态变化，而应该有“沉淀”的感觉。

比如：

- 卡片淡出
- 微缩
- 化成光点 / 纸屑 / 细线
- 时间线节点轻轻亮起
- 当前时间被标记

动效要轻，不要廉价烟花。

### 2.4 桌面空间感

桌面端应该像一个沉浸式工作台。

关键词：

- 全屏
- 大画布
- 无限缩放
- 横向时间线
- 空间层次
- 低干扰 UI

桌面端不是移动端放大版。

它应该负责复杂编排、关系整理、长时间使用。

### 2.5 移动快速捕捉

移动端的付费感不在复杂，而在快。

手机上不要强迫用户操作大画布。

移动端重点：

- 快速添加时间点
- 快速添加卡片
- 今日视图
- 底部 sheet
- 单手可操作
- 轻量回顾

移动端负责捕捉，桌面端负责编排。

## 3. 首批必须打磨的核心场景

### 场景 A：第一次打开

用户应该立刻理解：

- 这里可以记录时间线
- 这里可以创建卡片
- 卡片可以完成并进入时间线

不要用长教程。

可以用一个非常轻的空状态：

> 写下一个现在正在发生的事。

或者：

> 创建一张卡片，让它成为今天的起点。

### 场景 B：创建时间节点

桌面：点击时间线空白处，原地出现输入框。

移动：点击底部按钮，弹出底部 sheet。

提交后节点自然生长出来。

### 场景 C：创建 Todo 卡片

用户可以快速输入一件想做的事。

Todo 卡片和普通思维卡片视觉上要有轻微差别，但不要变成沉重任务管理器。

### 场景 D：完成 Todo

这是产品关键瞬间。

完成后：

1. 卡片消散。
2. 时间线生成节点。
3. 节点内容继承卡片文本。
4. 用户能感到“它被保存了”。

这应该成为 Now 时间线的招牌交互。

### 场景 E：回看一天 / 一周

用户回看时应该有一种“我的生活有形状”的感觉。

不是列表，不是报表，而是时间轨迹。

---

## 4. MVP 也要避免的廉价感

即使是 MVP，也尽量避免：

- 默认浏览器按钮风格
- 粗糙表单堆叠
- 后台管理系统布局
- 过多边框
- 过多颜色
- 复杂设置先行
- AI 配置暴露在首页
- 像工程 demo 一样的空状态

MVP 可以功能少，但不能气质散。

---

## 5. 产品顺序建议

不要先做 AI。

不要先做复杂同步。

不要先做设置页宇宙。

先做这三个付费级核心体验：

1. 时间线创建节点的手感
2. 画布创建 / 拖动卡片的手感
3. Todo 完成后转时间线的仪式感

只要这三个场景成立，产品就有骨架。

后面的 AI、报告、备份、同步，都是增强。

---

## 6. 一句话标准

如果用户看完只觉得“功能挺多”，那不够。

目标是让用户觉得：

> 我想把我的生活放进这里。

这才是 Now 时间线最应该打磨的付费感。
