# Now 时间线：作品定位与展示价值

## 1. 核心定位

Now 时间线可以先作为一个 **PWA 练手 / 求职作品级项目** 来推进。

它不需要一开始就被定义成商业产品，也不需要先纠结用户规模、收费方式或市场边界。

当前更重要的目标是：

> 做出一个完整、有审美、有交互记忆点、能展示产品思考和前端能力的作品。

它应该像一个可被展示的成品，而不是功能堆叠的 demo。

---

## 2. 作品集价值

这个项目适合作为求职 / 展示作品，因为它覆盖了很多前端能力点：

- PWA 应用架构
- 桌面端与移动端双端适配
- 复杂交互设计
- 拖拽画布
- 节点连线
- 时间线可视化
- 本地数据库 IndexedDB
- 动效设计
- 数据建模
- 可选 AI API 接入
- 可选备份 / 同步扩展

它不是一个普通 CRUD 项目。

如果完成度足够高，它可以展示：

> 我不只是会写页面，我能把一个抽象想法做成完整产品体验。

---

## 3. 可以对外表达的产品概念

不要把 Now 时间线描述成传统 Todo、日记或笔记软件。

更适合的表达是：

> 一个把想法、行动和生活记录连接起来的时间线画布。

或者更短一点：

> 用卡片计划今天，用时间线保存发生过的事。

核心差异点：

> 做完的事情，不是被划掉消失，而是变成时间线上的记录。

这是产品最容易被理解、也最有记忆点的地方。

---

## 4. 核心模块

### 4.1 时间线

时间线负责记录已经发生的节点。

它可以承载：

- 手动创建的时间点
- Todo 完成后自动生成的节点
- 一天 / 一周 / 一个月的回看
- 后续 AI 总结的来源数据

时间线不是任务列表，而是结果沉淀层。

### 4.2 画布

画布负责承载想法、计划和卡片关系。

它可以包含：

- 普通思维卡片
- Todo 卡片
- 卡片之间的连接
- 可拖拽、可缩放的空间布局

画布是行动发生前的组织空间。

### 4.3 Todo 卡片转时间线节点

这是核心交互闭环。

流程：

```txt
创建 Todo 卡片
  -> 完成 Todo
  -> 卡片消散
  -> 自动生成时间线节点
```

这个交互应该被重点打磨，因为它是 Now 时间线区别于普通 Todo 的关键瞬间。

---

## 5. AI 功能定位

AI 不是主功能，也不是 MVP 的地基。

它应该作为后续增强能力存在，主要用于：

- 周报
- 月报
- 年报
- 自定义时间段总结
- 从时间线节点中提炼主题

AI 的产品表达不要过度技术化。

用户看到的应该是：

> 生成回顾

而不是：

> 配置 Agent / 模型 / Token。

### 自定义 API

自定义 API 可以保留，但应该放在高级设置里。

它适合作为技术展示点：

- 支持 BYOK：Bring Your Own Key
- 支持自定义 endpoint
- 支持不同模型服务
- 展示系统的扩展性

但它不应该出现在主流程里。

---

## 6. 商业化暂不作为第一目标

当前不需要先围绕收费设计产品。

更稳的顺序是：

1. 先做出漂亮、顺滑、完整的 PWA 成品。
2. 让核心交互闭环成立。
3. 让它可以作为求职 / 作品集项目展示。
4. 再考虑是否开源、是否加 AI、是否做同步和备份。
5. 最后再讨论商业化可能性。

如果未来真的收费，卖点也不应该只是 AI Token。

更值得打磨的是：

- UI / UX
- 时间线视觉
- 画布手感
- Todo 完成仪式感
- 多端同步
- 自动备份
- 高级回顾

---

## 7. 当前优先级

第一阶段优先打磨：

1. 时间线创建节点
2. 画布创建卡片
3. 卡片拖拽和连线
4. Todo 完成后生成时间线节点
5. 桌面端与移动端的核心布局差异
6. PWA 安装体验

暂时不要让 AI、收费、账号系统、复杂同步抢走主线。

---

## 8. 一句话结论

Now 时间线当前最适合被定义为：

> 一个作品集级 PWA：用时间线和卡片画布，把想法、行动和发生过的事连接起来。

先按可展示、可开源、可求职的完成度做出来。

商业化和 AI 增强，是之后的扩展，不是现在的地基。
